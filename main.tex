\section{Objetivo}
 Desenvolver webpages dinâmicas, utilizando as linguagens HTML, CSS, PHP e estruturas em formato JSON para manejo de dados, com fim de mapear e exibir as atividades e grupos do ambiente da Escola de Engenharia da UFF.

\section{Motivação}
Principais motivações para a realização deste trabalho:
\begin{itemize}
    \item \textbf{Aperfeiçoamento Técnico:} Ao aplicar na prática o Modelo Cliente-Servidor, estudando e utilizando tecnologias de \textit{front-end} (HTML/CSS) e \textit{back-end} (PHP).
    \item \textbf{Organização e Estruturação:} Implementar, em um único catálogo  dinâmico e limpo, os dados dispersos sobre Associações, Coletivos, Empresas Juniores, etc..
    \item \textbf{Experiência do Usuário:} Fornecer uma navegação mais fluida e responsiva comparada a páginas estáticas tradicionais
    \item \textbf{Busca dos Dados:} O grande volume e dispersão de informações sobre os grupos da UFF resultam no exaustivo processo de coleta, filtragem e validação manual dos dados, exigindo um esforço considerável para torná-los confiáveis e consistentes.
\end{itemize}

\subsection{Introdução Teórica}

O projeto baseia-se no \textbf{Modelo Cliente-Servidor}, uma arquitetura organizada na divisão entre:
\begin{itemize}
    \item \textbf{Cliente:} o navegador, que solicita páginas ao servidor e exibe ao usuário.
    \item \textbf{Servidor:} a máquina que recebe os pedidos, processa os scripts PHP, consulta os arquivos JSON e devolve o HTML pronto.
\end{itemize}

Vale ressalatar que a \textbf{Internet} é a infraestrutura física que conecta as máquinas, enquanto a \textbf{World Wide Web (WWW)} é a operação que atua sobre essas infraestrutura através do sistema conectado por \textbf{hyperlinks}, que são acessíveis por meio das \textbf{URLs} e transferidos entre cliente e servidor pelo protocolo \textbf{HTTP}.\\

No \textit{front-end} (cliente) é usado o \textbf{HTML} para estruturar o conteúdo e o \textbf{CSS} para definir o visual e o layout. No \textit{back-end} (servidor), o \textbf{PHP} gera páginas dinamicamente, convertendo os dados armazenados em \textbf{JSON} em vetores e operando sobre eles para montar o HTML final.\\

Essa separação entre: camada de dados (arquivos JSON), lógica de processamento (PHP) e apresentação visual (HTML/CSS) atribui ao sistema \textbf{modularidade} e \textbf{manutenibilidade}.

\subsection{Lógica de Processamento}

\paragraph{1. Matriz de Categorias}
Um vetor para mapear o título de cada bloco ao caminho do seu arquivo de dados em formato JSON. O que garante manutenibilidade e escalabilidade:

\begin{lstlisting}[language=php]
$categorias = [
    "Associações"            => "dados/associacoes.json",
    "Coletivos"              => "dados/coletivos.json",
    "Empresas Juniores"      => "dados/empresas_juniores.json",
    "Organizações"           => "dados/organizacoes.json",
    "Órgãos Representativos" => "dados/orgaos_representativos.json",
    "Projetos"               => "dados/projetos.json"
];
\end{lstlisting}
    
\paragraph{2. Verificação de Existência}
Apenas como medida de segurança, a função \ \texttt{file\_exists()} para impedir falhas caso algum arquivo seja corrompido ou removido:

\begin{lstlisting}[language=php]
if (file_exists($caminho_arquivo)) {
    // O processamento só ocorre se o arquivo existir
}
\end{lstlisting}
\newpage
\paragraph{3. Alocação de Memória}
O conteúdo é convertido em uma lista, permitindo o acesso indexado:

\begin{lstlisting}[language=php]
$json_puro = file_get_contents($caminho_arquivo);
$grupos = json_decode($json_puro, true);
\end{lstlisting}
    
\paragraph{4. Laço Duplo}
Dois laços de repetição \ \texttt{foreach} alinhados para varrer as categorias e, em seguida, gerar os cartões visuais para cada grupo:

\begin{lstlisting}[language=php]
foreach ($categorias as $titulo_bloco => $caminho_arquivo) {
    if (file_exists($caminho_arquivo)) {
        if($grupos != null){
            foreach ($grupos as $grupo) {
                // Renderização os Cards no layout HTML
            }
        }
    }
}
\end{lstlisting}
    
\paragraph{5. Tratamento de Inconsistências}
Validar a existência do endereço web e verificar se o texto ja inicia com \texttt{"https://"} pois, se não, implementa automaticamente.

\begin{lstlisting}[language=php]
$site = isset($grupo['website']) ? trim($grupo['website']) : '';

if (!empty($site) && $site != "Não informado" && $site != "Não possui") {
    if (strpos($site, 'http://') !== 0 && strpos($site, 'https://') !== 0) {
        $link_destino = "https://" . $site;
    } else {
        $link_destino = $site;
    }
    $classe_css = "card";
    $target = "target='_blank'";
} else {
    $link_destino = "#";
    $classe_css = "card sem-link";
    $target = "";
}
\end{lstlisting}

\subsection{Método}
Etapas para obter os resultados:
\begin{enumerate}
    \item \textbf{Levantamento de Dados:} Pesquisa manual em sites e redes sociais sobre os grupos da UFF, categorizando as informações pedidas (Nome, Morada, E-mail, etc..)
    \item \textbf{Estrutura da Base de Dados:} Criação de arquivos \texttt{.json} separados por categorias
    \item \textbf{Visual:} Criação de um layout em HTML/CSS com propriedades flexíveis (\texttt{display: flex}) e focando em aplicar a paleta de cores do PET-Tele.
    \item \textbf{Web Dinâmica:} Escrita em PHP para processar os JSONs e alocar os dados nos blocos visuais (Cards).
\end{enumerate}
